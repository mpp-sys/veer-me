# Veer.me — Claude Code Build Instructions

Veer.me (Dare to change direction. Reinvent with courage.) is an AI-powered coaching and career companion app. It helps knowledge workers whose jobs are at risk from AI to reinvent themselves — using their skills, talents, interests, and networks — with clarity, courage, and a structured plan.

The companion AI inside Veer.me is called **Shift**.

---

## 🎯 Prototype Goal

## 🌀 What is Veer.me?

Build a clickable prototype with real AI conversation suitable for investor demos. The prototype must:

* Require no sign-in (guest account only)
* Be fully functional as a web app (mobile-responsive)
* Demonstrate the core user journey end-to-end
* Use the Anthropic API (claude-sonnet-4-6) to power Shift's coaching conversations
* Feel polished and investor-ready — not a rough wireframe

---

## 🛠️ Tech Stack

* **Frontend:** React + Tailwind CSS
* **Backend:** Node.js / Express (or Next.js full-stack)
* **AI:** Anthropic API (claude-sonnet-4-6) via `@anthropic-ai/sdk`
* **State:** Local state only for prototype (no database needed)
* **Styling:** Bold & modern, gradient & dynamic (deep dark backgrounds, bright accent colours — think Perplexity / Superhuman)
* No auth required for prototype

## 🗺️ User Journey to Build

Build these screens/flows in order. Each is a milestone.

### Screen 1 — Landing / Intro

* Hero screen with Veer.me branding
* Tagline: "You're not alone in this."
* Short text explaining what Veer.me does (2–3 sentences max)
* Single CTA button: "Start your reinvention"
* Visual metaphor: a person at a crossroads — journey and choice
* Brand colours: deep gradient (dark navy → electric blue → violet), white text
* NO video for prototype — use animated text or a bold static illustration

### Screen 2 — Situation Assessment (Onboarding Step 1)

* Shift introduces itself warmly: "Hi, I'm Shift — your reinvention companion. I'm here to help you navigate the AI revolution. Let's start by understanding where you are right now."
* Ask the user: "What's bringing you here today?" with 3 cards to choose from:
  * 🧭 My job feels at risk from AI
  * 🚀 I want to reinvent my career entirely
  * 💡 I want to use AI as an opportunity, not a threat
* User selects one card (single select)
* Progress indicator at top (Step 1 of 4)

### Screen 3 — Time Horizon Selection (Onboarding Step 2)

* Shift responds to the user's choice with a short empathetic message (use real AI here — call the Anthropic API with the user's selection)
* Then asks: "How urgent is this for you?" with 3 cards:
  * ⚡ Intervention — I need help now (1 month)
  * 📅 Transition — I'm planning ahead (6 months)
  * 🌱 Reinvention — I'm thinking long-term (12 months)
* User selects one
* Progress indicator: Step 2 of 4

### Screen 4 — Deep Profiling (Onboarding Step 3)

* Shift says: "To give you the best guidance, I need to get to know you. Answer honestly — everything you share stays completely private."
* Show a structured form with the following fields (keep it light for prototype — not overwhelming):
  * Current role / what you do (text input)
  * Your biggest strength (text input)
  * What energises you most at work (text input)
  * What you're most worried about (text input)
* Progress indicator: Step 3 of 4
* CTA: "Meet Shift"

### Screen 5 — Shift Intro + First Coaching Message (Onboarding Step 4)

* Full-screen moment: "Shift has read your profile."
* Shift delivers a personalised first message (REAL AI call to Anthropic API)
* The message should:
  * Acknowledge what the user shared
  * Name one strength it noticed
  * Offer a reframe (turn fear into agency)
  * End with one concrete question to start the coaching
* Progress indicator: Step 4 of 4 ✓
* CTA: "Start talking to Shift" → takes user to the main app

### Screen 6 — Main App Dashboard

The main app has two areas, always visible:

**Left panel / Top: Shift Chat (persistent)**

* Always-available chat interface with Shift
* Chat is powered by real Anthropic API calls
* Shift maintains context of the user's profile and selected path throughout the session
* Shift's tone: expert guide — authoritative, precise, warm. NOT a best friend.
* Shift should NEVER give generic advice. Always reference what the user shared.

**Right panel / Bottom: Structured Dashboard**

* Show these 4 elements as cards:
  * **My Reinvention Path** — shows their chosen horizon (1/6/12 months) with a simple visual timeline
  * **This Week's Focus** — one concrete action Shift has suggested (generated by AI)
  * **Progress Check** — a simple wellbeing/progress tracker with 3 quick questions (1–5 scale):
    * "How clear do you feel about your direction?"
    * "How motivated are you this week?"
    * "How supported do you feel?"
  * **Milestone** — a celebration card when the user completes something (for prototype: show a sample milestone like "You've started your reinvention journey 🎉")

## 🤖 Shift — AI Persona & System Prompt

When calling the Anthropic API, always use this system prompt for Shift:

```
You are Shift, the AI coaching companion inside Veer.me (Dare to change direction. Reinvent with courage.).
Your role is to help knowledge workers reinvent themselves and navigate the AI revolution with clarity, courage, and a concrete plan.

Your personality:
- Calm and grounding — you never amplify anxiety
- Bold and energising — you don't shy away from hard truths or big possibilities
- Warm and empathetic — you see the human behind the career challenge
- Sharp and insightful — you bring precision, not vague advice

Your voice:
- You speak like an expert guide: authoritative, precise, and reassuring
- You are NOT a best friend or therapist
- You are professional yet warm
- You never give generic advice — always reference what the user has shared
- You ask one focused question at a time
- You keep responses concise (max 150 words per message)

Responsible AI rules you must follow:
- Never push the user toward a specific decision — present options, not prescriptions
- If a user shows signs of distress (not just uncertainty), acknowledge it and suggest speaking to a human professional
- Never encourage dependency on you — remind users that human connection matters
- Maintain privacy — never reference user data in ways that feel surveillance-like
- Do not make medical, legal, or financial recommendations

User context will be provided at the start of each session. Always use it.
```

## 🔌 Anthropic API Integration

```js
// src/hooks/useShift.js
import Anthropic from '@anthropic-ai/sdk';

const client = new Anthropic({
  apiKey: import.meta.env.VITE_ANTHROPIC_API_KEY,
  dangerouslyAllowBrowser: true // prototype only
});

export async function callShift(messages, userProfile) {
  const systemPrompt = buildSystemPrompt(userProfile);

  const response = await client.messages.create({
    model: 'claude-sonnet-4-6',
    max_tokens: 400,
    system: systemPrompt,
    messages: messages
  });

  return response.content[0].text;
}

function buildSystemPrompt(userProfile) {
  return `[SHIFT SYSTEM PROMPT — paste full prompt here]

User Profile:
- Situation: ${userProfile.situation}
- Time horizon: ${userProfile.horizon}
- Current role: ${userProfile.role}
- Biggest strength: ${userProfile.strength}
- What energises them: ${userProfile.energiser}
- What they're worried about: ${userProfile.worry}`;
}
```

## ✅ Build Order & Milestones

Build in this exact order. Test each screen before moving to the next.

1. **Project setup** — Vite + React + Tailwind + Anthropic SDK + `.env`
2. **Design system** — colours, typography, base components (Button, Card, SelectionCard)
3. **Landing screen** — fully styled, animated gradient background
4. **Onboarding flow** — Screens 2, 3, 4 (no AI yet, just UI + state)
5. **Anthropic integration** — wire up `useShift.js`, test with a dummy call
6. **Shift Intro screen** — first real AI call, personalised message
7. **Dashboard** — layout, Reinvention Path card, Progress tracker
8. **Shift Chat** — persistent chat panel with full conversation memory
9. **Milestone card** — sample milestone display
10. **Polish pass** — animations, transitions, mobile responsiveness
11. **Demo prep** — seed a sample user profile so the demo starts mid-journey

## ⚠️ Important Rules

* **Never commit `.env`** — add it to `.gitignore` immediately
* **No real user data** in the prototype — use fake/demo profiles only
* **Keep it simple** — this is an investor demo, not a production app. Fake data is fine.
* **One file per component** — keep components small and focused
* **No auth** — skip login/signup entirely for the prototype
* **API key in `.env`** as `VITE_ANTHROPIC_API_KEY`
* **Mobile responsive** — the demo may be shown on a phone
* **Test every AI call** — make sure Shift's responses are always relevant, not generic
